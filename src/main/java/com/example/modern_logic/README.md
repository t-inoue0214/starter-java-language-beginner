# 第6章: 現代的な書き方（StreamとOptional）

Java 8（2014年）以降、Javaの書き方は劇的に進化しました。
現場では必ず使われる2つの「モダンな武器」を授けます。

## 1. Stream API (ストリーム) - 全自動ベルトコンベア

これまでの `for` 文は、自分で「1つ取り出して、ifでチェックして、新しいリストに入れて…」と手作業をする感じでした。
Stream APIは、**「データのベルトコンベア」**です。

* **流れ:**
    1. **材料を流す** (`stream()`)
    2. **選別する** (`filter`): 「腐ったリンゴは弾く」
    3. **加工する** (`map`): 「皮を向く」
    4. **箱詰めする** (`collect`): 「出荷用の箱にまとめる」

「やり方（How）」ではなく「やりたいこと（What）」を書くので、コードが物語のように読めるようになります。

## 2. Optional (オプショナル) - 空っぽかもしれない箱

`null` はプログラムをクラッシュさせる危険な存在です。
Optionalは、**「中に値が入っているかもしれないし、空っぽかもしれない箱」**です。

* **昔:** 変数を裸で渡す。「中身ある？ぬるぽ（NullPointerException）！」
* **今:** 箱(`Optional`)に入れたまま渡す。「箱を開けてみて、もしあったら使う。なければ予備を使う」

この箱を使うことで、「値がない場合」を強制的に意識させられるため、安全性が飛躍的に向上します。
