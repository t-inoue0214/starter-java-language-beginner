# 第1章: プログラムの流れを操ろう（制御構文とメソッド）

ようこそ！ここでは、プログラムの動きを自由自在に操るための「3つの魔法」を覚えます。

1. **分岐 (Branching):** 条件によって違うことをする。
2. **反復 (Loop):** 同じことを繰り返す。
3. **メソッド (Method):** 処理をまとめて名前をつける。

## 1. 分岐 (If / Switch)

プログラムは通常、上から下へ一本道で進みますが、「もし明日が晴れなら洗濯をする、雨なら読書をする」のように、状況に合わせて動きを変えたいときがあります。
これを実現するのが `if` と `switch` です。

* **if:** 「もし〜なら」という条件が細かいときに使います（例：テストの点が60点以上なら合格）。
* **switch:** 「サイコロの目が1ならこれ、2ならこれ」のように、値がカチッと決まっているときに使います。

## 2. 反復 (For / While)

「100回スクワットする」という処理を書くとき、`System.out.println("スクワット！");` を100行書くのは大変ですよね？
「〜の間、繰り返す」という命令を使えば、たった数行で書けます。

* **for:** 「10回繰り返す」のように、回数が決まっているときに使います。
* **while:** 「体力がなくなるまで繰り返す」のように、回数が決まっていないときに使います。

## 3. メソッド (Method)

メソッド（Method）とは、目的を達成するための「効率的で体系的な手法・手順」のことです。

ちなみに、フレームワークという言葉がありますが、以下のような違いがあります。

* フレームワークは複雑な課題を整理・構造化するための「思考の枠組み（地図）」
* メソッドは具体的な手順やテクニックを指す「解決手法（羅針盤）」

### なぜメソッドを作るの？

毎回ゼロから料理を作るのは大変ですよね。

「カレー」を作る場合、カットの仕方や食材を入れる順番と量、煮込む時間にパターンがあります。

「カレー」というレシピ（メソッド）があれば、「カレー作って！」と頼むだけで済みます。

プログラムも同じで、よく使う処理をメソッドにしておけば、いつでも呼び出すだけで実行できます。

### メソッドの構成要素（料理に例えると…）

メソッドを作るときは、以下の3つを決めます。

1. **引数（ひきすう）**：料理人に渡す「材料」。
   * 例：「豚肉」と「野菜」を渡す。
2. **処理**：実際にすること。
   * 例：切って煮込む。
3. **戻り値（もどりち）**：完成してカウンターに出される「料理」。
   * 例：美味しいカレーライス。

> **Note:**
> 完成品が必要ない（ただ挨拶するだけ、画面に表示するだけ）場合は、戻り値なしの意味である **`void`（ボイド）** を使います。
