# 第3章: データをまとめて管理しよう（コレクション）

変数は「箱」でしたが、データが100個あったら箱を100個用意するのは大変です。

Javaには、データを並べる方法が大きく2つあります。
**「昔ながらの配列」** と、**「便利なリスト」** です。

この2つは日本語だとどちらも「配列」と呼ばれることがあって紛らわしいですが、**機能は全く別物** です。

## 1. 配列 (Array) - サイズ固定のコインロッカー

* **コード:** `String[]` や `int[]`
* **特徴:** 作るときに「3個入る」と決めたら、一生3個のままです。
* **弱点:** データが増えたときに困ります。あと、便利な機能（検索や並べ替え）がついていません。
* **用途:** データの個数が絶対に変わらないとき（例：1週間の曜日＝7個）に使います。

## 2. List (リスト) - 伸び縮みする魔法のバッグ

* **コード:** `List<String>` (`ArrayList`)
* **特徴:** データを入れると勝手に大きくなり、消すと縮みます。
* **正体:** 実は、Listの裏側では「配列」が動いています。「配列の面倒な操作（サイズ変更など）」を全自動でやってくれる **便利なロボット** だと思ってください。
* **用途:** 現場での開発は、ほぼこちらを使います。
* **順序:** 入れた順に並びます。「0番目、1番目…」と番号（インデックス）で管理します。つまり、当たり前ですがデータを入れたときの順序を保証します。

## 3. Map (マップ) - 検索できる辞書

* **代表選手:** `HashMap`
* **特徴:** 「番号」ではなく「キーワード（キー）」でデータを管理します。
* **イメージ:** 辞書（単語→意味）、住所録（名前→住所）、ログイン情報（ID→パスワード）。
* **構造:** Key（鍵）と Value（値）のペアで保存します。Map自体は順序を保証していませんので、Key名を1〜10にしても、通常では順番どおりにはなりません。

## 4. ジェネリクス `< >` とは？

ListやMapを使うとき、`<String>` や `<Integer>` という表記が出てきます。
これは **「この袋には何を入れるか」を指定するラベル** です。

* `List<String>`: 文字列専用のリスト。数字を入れようとするとエラーにして守ってくれます。
